## 4. 레거시 부수 수식어

조합 테이블에 매칭되지 않는 작은 스트로크는 **형태**로 분류됩니다.

| 조건 | 수식어 | 의미 |
|------|--------|------|
| 가로·세로 < 15px | `[응축된]` | 점 (dot) |
| 가로 > 세로×2 | `[차단된]` | 수평 막대 |
| 세로 > 가로×2 | `[탈취의]` | 수직 고리 |
| 그 외 | `[변이된]` | 곡선/파동 |

> 메인 룬 bbox의 15% 미만인 스트로크만 부수로 분리됩니다.

## 6. 역학 (Dynamics) 분류

필기 속도에 따라 자동 판정됩니다.

| 평균 속도 | 역학 | 마법 성질 |
|-----------|------|-----------|
| < 0.3 px/ms | 느림(응축) | 지속형 · 방어형 |
| 0.3 ~ 1.5 px/ms | 안정적(지속) | 균형형 |
| > 1.5 px/ms | 거침(폭발적) | 폭발형 · 순간형 |

> 보조 도구(직선 자/컴퍼스) 사용 시 강제로 느린 필기로 처리 (t - 5000ms hack)

## 7. 열역학 시스템

```
공명(Resonance) = min(뼈대 획수 × 12.5, 100) Hz
열(Heat)        = Σ(speed × 500) + (부수 수 × 50)
밀도(Density)   = 룬 스트로크 총 포인트 수
부피(Volume)    = 뼈대 bbox 면적 / 10000 (최소 1)
압력(Pressure)  = (밀도 × max(열, 1)) / (부피 × 10)
불안정성        = clamp(압력×0.1 + 열×0.05 - 공명×0.5 + 조합보너스, 0, 100)

오버로드 조건: 열 > 바탕재 최대열 OR 불안정성 ≥ 100
```

## 8. 인식 엔진 메모

- **알고리즘**: $P Point-Cloud Recognizer
- **리샘플링**: 32 포인트로 정규화
- **스케일**: Anisotropic (가로/세로 독립 정규화, 축퇴 시 uniform fallback)
- **매칭 임계값**: `0.95` (거리 기준, 낮을수록 유사)
- **3-pass 분석**:
  1. **Compound pass**: 클러스터링 또는 leave-one-stroke-out으로 2개 룬 분리 → 조합 테이블 조회
  2. **Holistic pass**: 전체 스트로크를 하나의 룬으로 인식 시도
  3. **Radical strip pass**: 작은 스트로크 제거 후 나머지로 인식
- **클러스터링 패딩**: 전체 bbox의 6% (가까운 스트로크 합침, 분리된 부수 유지)

## 9. 문장 규칙 — 배치법 (Arrangement)

단어(개별 룬)를 넘어 여러 룬을 **공간적으로 배치**하면 "문장"이 됩니다.
배치 형태에 따라 마법의 성질과 위력이 근본적으로 달라집니다.

| 배치법 | 형태 | 효과 | 위력 배율 | 불안정성 보정 |
|--------|------|------|-----------|---------------|
| **직선 배열** (Linear) | `A ─ B ─ C` | 순차 시전. 각 룬이 순서대로 발동, 연쇄 효과 | ×1.0 | +0 |
| **원형 배열** (Circular) | 뼈대 원 위에 룬 배치 | 지속/증폭. 마력이 순환하며 자기 강화 | ×1.5 | +15 |
| **삼각 배열** (Triangular) | 뼈대 삼각형 꼭짓점에 룬 3개 | 안정 균형. 세 원소가 서로 견제 | ×1.2 | -10 |
| **방사형 배열** (Radial) | 중심에서 바깥으로 퍼지는 배치 | 광역 방출. 범위가 넓어지나 집중도↓ | ×0.8 (개당) | +20 |
| **중첩 배열** (Overlapping) | 같은 위치에 겹쳐 그리기 | 융합/합성. 현재 부수 조합 시스템 | ×2.0 | 조합표 참조 |
| **대칭 배열** (Symmetric) | 중심축 기준 좌우 대칭 | 공명 보너스. 대칭도에 비례하여 안정 | ×1.3 | -15 |

### 배치법 세부 규칙

**직선 배열**
- 룬 간 간격이 좁을수록 연쇄 속도 ↑ (간격 < 뼈대 bbox 20% → "밀집 직선")
- 3개 이상 직선 배열 시 "연쇄술식"으로 판정
- 예: `이사(|) ─ 열기(△) ─ 소울로(⚡)` = 얼음→불→번개 순차 폭발

**원형 배열**
- 뼈대 원(○)을 먼저 그린 뒤, 원 둘레에 룬을 배치
- 룬 수가 많을수록 순환 마력 증가 (n × 0.3 배율 추가)
- 원 내부에 추가 룬 = "핵" 역할 → 순환 마력이 핵으로 집중
- 예: 원 둘레에 `이사 · 이사 · 이사` + 핵에 `열기(△)` = 빙벽으로 둘러싼 화염핵

**삼각 배열**
- 뼈대 삼각형 꼭짓점에 정확히 3개의 룬
- 세 룬의 원소가 모두 다르면 "삼원소 균형" 보너스 (불안정성 -20)
- 같은 룬 3개 = "삼중 강화" (위력 ×2.0, 불안정성 +30)

## 10. 문장 규칙 — 뼈대 상호작용 (Bone Interaction)

뼈대(Bone 모드)는 단순 수식어를 넘어, **룬과의 공간 관계**에 따라
마법의 방향·범위·성질을 결정하는 "문법 틀"입니다.

### 상호작용 유형

| 유형 | 공간 관계 | 효과 | 시각 표현 |
|------|-----------|------|----------|
| **가두기** (Enclosing) | 뼈대가 룬을 완전히 둘러쌈 | 에너지 억제/집중. 내부 위력↑, 외부 영향↓ | 뼈대 글로우 강화 |
| **걸치기** (Crossing) | 뼈대가 룬 위를 관통 | 에너지 분할/불안정. 위력↑ but 불안정성↑↑ | 교차점 스파크 |
| **받치기** (Underlying) | 뼈대를 먼저 그리고 위에 룬 | 안정화 기반. 불안정성↓, 지속 시간↑ | 뼈대 부드러운 글로우 |
| **연결** (Bridging) | 뼈대가 두 룬 사이를 이음 | 마력 흐름 통로. 연결된 룬 간 에너지 전달 | 마력선 흐르는 효과 |
| **관통** (Piercing) | 뼈대 직선이 룬 중심을 꿰뚫음 | 방향성 부여. 마법이 직선 방향으로 발사 | 뼈대 끝에서 발사체 |
| **감싸기** (Wrapping) | 뼈대 나선/곡선이 룬을 감쌈 | 에너지 응축. 느리지만 강력한 단발 | 나선 수축 애니메이션 |

### 가두기 (Enclosing) 세부

뼈대 형태에 따라 가두기 효과가 달라집니다:

| 뼈대 형태 | 가두기 효과 | 위력 보정 | 불안정성 보정 |
|-----------|------------|-----------|---------------|
| 원(○) | **봉인** — 전방위 균일 억제 | ×1.2 | -10 |
| 삼각형(△) | **집중** — 꼭짓점 방향으로 에너지 집중 | ×1.5 (방향) | +5 |
| 사각형(□) | **결계** — 완전 차단/보호 | ×0.8 | -25 |
| 마름모(◇) | **공명** — 내부 반사로 자기 증폭 | ×1.8 | +20 |

### 걸치기 (Crossing) 세부

뼈대가 룬을 가로지르는 각도와 방식:

| 걸침 방식 | 효과 | 불안정성 |
|-----------|------|----------|
| 수평 관통 (ㅡ) | 에너지 수평 분산 | +10 |
| 수직 관통 (\|) | 에너지 상하 분리 | +15 |
| 대각선 관통 (/) | 에너지 비대칭 분할 → 예측 불가 | +25 |
| 십자 관통 (+) | 사방 분열 → 광역화 but 약화 | +30 |

### 연결 (Bridging) 세부

뼈대로 두 룬을 이으면 마력이 한쪽에서 다른 쪽으로 흐릅니다.
연결선의 **형태·굵기·수·패턴**에 따라 전달 방식이 크게 달라집니다.

#### 연결선 유형

| 연결선 | 그리기 | 전달 속도 | 전달 효율 | 특수 효과 | 불안정성 |
|--------|--------|-----------|-----------|-----------|----------|
| **단선** (─) | 직선 1개 | 빠름 | 90% | 없음 (기본) | +0 |
| **이중선** (═) | 평행 직선 2개 | 빠름 | 100% (무손실) | 양방향 흐름 가능 | +5 |
| **삼중선** (≡) | 평행 직선 3개 | 매우 빠름 | 120% (증폭) | 과충전 전달, 수신 룬 위력↑ | +15 |
| **점선** (···) | 끊어진 짧은 선 | 느림 | 50% | 간헐적 전달, 깜빡임 효과 | -5 |
| **파선** (- -) | 긴-짧 반복 | 보통 | 70% | 필터링: 불안정 에너지만 차단 | -10 |
| **물결선** (~) | 곡선 파동 | 느림 | 80% | 에너지 축적, 방출 시 폭발적 | +10 |
| **나선** (⌀) | 나선형 감기 | 매우 느림 | 60% | 응축 전달, 도착 시 ×2.0 폭발 | +20 |
| **갈래선** (⑂) | 중간에서 분기 | 빠름 | 각 50% | 1→2 분배 (다중 대상) | +5 |
| **고리선** (∞) | 양 끝이 루프 | 순환 | 유지 | 마력 순환, 자기 유지 회로 | +15 |

#### 연결선 세부 규칙

**단선 (─) — 기본 통로**
- 가장 단순한 마력 전달 경로
- 길이에 비례하여 미세한 손실 발생 (10px당 1% 감소)
- 직선일수록 효율적, 꺾이면 꺾인 지점에서 에너지 누출
- 예: `열기(△) ─── 알기즈(Y)` = 불 에너지가 방패로 전달 → 화염 방벽

**이중선 (═) — 쌍방향 통로**
- 두 룬 사이 에너지가 **양방향**으로 흐름
- 두 룬이 서로의 속성을 **교환**
- 무손실 전달 → 안정적인 장기 연결에 적합
- 예: `이사(|) ═══ 열기(△)` = 얼음과 불이 지속 교환 → 증기 생성 루프
- **그리기**: 두 직선을 가까이 평행하게 그림 (간격 < 선 길이의 10%)

**삼중선 (≡) — 과충전 통로**
- 에너지가 **증폭**되며 전달 (120%)
- 수신 룬의 위력이 1.2배로 강화됨
- 과부하 위험 → 불안정성 증가
- 지속 사용 시 연결선 자체가 과열 (시각: 붉은 글로우)
- 예: `소울로(⚡) ≡≡≡ 티와즈(↑)` = 번개로 과충전된 투창
- **그리기**: 세 직선을 가까이 평행하게 그림

**점선 (···) — 간헐 통로**
- 에너지가 **끊어졌다 이어졌다** 반복
- 맥박처럼 주기적으로 전달 → 타이밍 제어에 유용
- 불안정 에너지를 자연 감쇠시킴 (안정화 효과)
- 예: `열기(△) ····· 나우디즈(+)` = 불을 약하게 간헐적으로 봉인에 주입
- **그리기**: 짧은 선분을 간격 두고 여러 개 그림

**물결선 (~) — 축적 통로**
- 에너지가 물결을 따라 **축적**되며 이동
- 도착 시점에 축적된 에너지가 한꺼번에 방출
- 물결이 클수록(진폭 큼) 축적량 ↑, 전달 속도 ↓
- 예: `이사(|) ~~~ 소울로(⚡)` = 냉기를 천천히 모아 번개로 한 번에 전환
- **그리기**: 자유 곡선으로 물결 형태

**나선 (⌀) — 응축 통로**
- 에너지가 나선을 돌며 극도로 **압축**
- 도착 시 응축된 에너지가 ×2.0 배율로 폭발
- 가장 느리지만 가장 강력한 단발 전달
- 나선 회전수가 많을수록 응축 ↑ (최대 5회전 = ×3.0)
- 예: `열기(△) ⌀⌀⌀ 케나즈(<)` = 불을 극한 압축 후 횃불로 발사
- **그리기**: 룬 사이를 나선형으로 감으며 연결

**갈래선 (⑂) — 분배 통로**
- 하나의 출발 룬에서 **여러 도착 룬**으로 분배
- 각 갈래에 균등 분배 (2갈래 = 각 50%, 3갈래 = 각 33%)
- 갈래 굵기(그리기 속도)로 비율 조절 가능
- 예: `소울로(⚡) ⑂─ 알기즈(Y)` + `⑂─ 티와즈(↑)` = 번개를 방패와 창에 동시 분배
- **그리기**: 직선을 그리다가 중간에서 두 방향으로 분기

**고리선 (∞) — 순환 통로**
- 양 끝이 각각 룬을 감싸며 **폐회로** 형성
- 마력이 무한 순환 → 자기 유지 (시전 후 수동 해제 필요)
- 순환 속도가 빨라질수록 위력 ↑, 불안정성 ↑
- 오래 유지하면 과열 → 오버로드 위험
- 예: `열기(△) ∞∞ 열기(△)` = 불끼리 무한 순환 → 화구(火球) 자기유지
- **그리기**: 8자 또는 루프 형태로 두 룬을 감싸며 연결

#### 연결 교차점 (Junction) 규칙

여러 연결선이 만나는 지점에서의 동작:

| 교차 유형 | 형태 | 효과 |
|-----------|------|------|
| **통과 교차** | 두 연결선이 겹치지만 연결 안 됨 | 서로 간섭 없음 (높이가 다른 것으로 취급) |
| **합류 교차** | 두 연결선이 한 점에서 만남 | 에너지 합산, 합류점에서 혼합 |
| **충돌 교차** | 반대 방향 흐름이 만남 | 에너지 상쇄 → 폭발 or 소멸 (원소에 따라) |

#### 연결 방향성

> **기본 규칙**: **먼저 그린 쪽(출발) → 나중에 그린 쪽(도착)**으로 마력 흐름
>
> 이중선(═)만 예외적으로 양방향 허용

| 흐름 방향 | 판정 기준 | 시각 표현 |
|-----------|-----------|-----------|
| 출발 → 도착 | 뼈대 그리기 시작점 → 끝점 | 대시 애니메이션 방향 |
| 양방향 (═만) | 이중선 사용 시 자동 | 양방향 대시 교차 |
| 역류 | 불가 (강제 역류 시 불안정성 +50) | 붉은 스파크 경고 |

## 11. 문장 규칙 — 접속사와 조사 (Connectors)

특정 룬은 두 룬 사이에 놓일 때 **접속사(조사)** 역할을 합니다.
이때 접속사 룬 자체의 마법은 발동하지 않고, 문법적 기능만 수행합니다.

### 접속사 룬

| 접속사 룬 | 기능 | 의미 | 사용 예 |
|-----------|------|------|----------|
| 대지(ㅡ) | **병렬 (AND)** | 양쪽 룬이 동시 발동 | `열기 ㅡ 이사` = 불+얼음 동시 |
| 이사(\|) | **순차 (THEN)** | 왼쪽 먼저, 오른쪽 나중 | `열기 \| 소울로` = 불 → 번개 |
| 게보(X) | **변환 (INTO)** | 왼쪽이 오른쪽으로 변환 | `이사 X 열기` = 얼음이 불로 변환 |
| 나우디즈(+) | **속박 (BIND)** | 양쪽을 하나로 고정/융합 | `알기즈 + 열기` = 방패에 불 속박 |
| 케나즈(<) | **지향 (TOWARD)** | 꺾쇠 방향으로 에너지 지향 | `열기 < 대상` = 불을 대상에게 |
| 다가즈(◇) | **전환 (SHIFT)** | 상태 전이/모드 변경 | `이사 ◇ 라구즈` = 얼음→물 전이 |

### 접속사 판정 조건

접속사로 인식되려면:
1. 두 메인 룬 **사이**에 위치 (bounding box 기준)
2. 메인 룬보다 **작거나 같은** 크기
3. 좌우 또는 상하 룬과 **겹치지 않음** (겹치면 부수 조합으로 판정)

### 조사 체계 (Particle System)

메인 룬에 붙는 **소형 획(장식)**이 문법적 역할을 합니다.
한국어의 조사(은/는/이/가/을/를), 영어의 be동사/전치사, 부사 등에 해당합니다.

#### 1) 격조사 — 역할 표시 (Case Markers)

룬이 문장에서 어떤 역할인지 표시합니다.

| 조사 | 그리기 | 위치 | 역할 | 의미 | 사용 예 |
|------|--------|------|------|------|---------|
| **주격** (이/가) | 점 1개 (·) | 룬 좌상단 | 주어 표시 | 이 룬이 행위의 주체 | `·열기 + 티와즈` = **불이** 솟구친다 |
| **목적격** (을/를) | 점 1개 (·) | 룬 우하단 | 목적어 표시 | 이 룬이 행위의 대상 | `이사 + 열기·` = 얼음이 **불을** 향한다 |
| **처격** (에/에서) | 짧은 가로선 (—) | 룬 아래 | 장소/기반 | 이 룬이 발동 위치 | `열기 + —라구즈` = 불이 **물에서** 일어남 |
| **구격** (으로/로) | 짧은 대각선 (/) | 룬 우측 | 수단/도구 | 이 룬을 수단으로 사용 | `알기즈 + /이사` = 방패를 **얼음으로** 만듦 |
| **여격** (에게) | 짧은 세로선 (\|) | 룬 우측 | 대상/수신자 | 에너지가 향하는 대상 | `열기 + \|알기즈` = 불을 **방패에게** |
| **공동격** (와/과) | 점 2개 (··) | 룬 양옆 | 함께/동반 | 양쪽이 결합하여 행동 | `··열기 + ··이사` = 불**과** 얼음 |

#### 2) 강도 부사 — 증폭과 약화 (Intensity Modifiers)

마법의 세기를 조절합니다. 메인 룬 주변에 **반복 획**으로 표시합니다.

| 부사 | 그리기 | 위치 | 효과 | 위력 배율 | 불안정성 |
|------|--------|------|------|-----------|----------|
| **미약** (살짝) | 점 1개 아래 | 룬 아래 | 최소 출력 | ×0.3 | -15 |
| **약화** (조금) | 짧은 가로선 아래 | 룬 아래 | 약한 출력 | ×0.5 | -10 |
| *(기본)* | 없음 | — | 표준 출력 | ×1.0 | ±0 |
| **강화** (매우) | 짧은 세로선 1개 | 룬 위 | 강한 출력 | ×1.5 | +10 |
| **극대** (몹시) | 짧은 세로선 2개 (\|\|) | 룬 위 | 매우 강한 출력 | ×2.0 | +25 |
| **극한** (절대적) | 짧은 세로선 3개 (\|\|\|) | 룬 위 | 최대 출력 | ×3.0 | +50 |
| **폭주** (한계초월) | 룬 위 작은 X 표시 | 룬 위 | 제어 불능 폭발 | ×5.0 | +90 |

> **규칙**: 강도 부사는 **누적 불가**. 하나의 룬에 하나의 강도만 적용.
> 폭주(×5.0)는 거의 확실히 오버로드를 일으킴 — 의도적 자폭 용도.

#### 3) 시제/상 표시 (Tense & Aspect)

마법의 발동 타이밍과 지속 방식을 조절합니다.

| 시제/상 | 그리기 | 위치 | 효과 |
|---------|--------|------|------|
| **즉발** (현재) | 없음 (기본) | — | 즉시 발동, 단발 |
| **지연** (미래) | 작은 원 (○) | 룬 앞(좌) | 일정 시간 후 발동 (축적 후 방출) |
| **지속** (진행) | 물결선 (~) | 룬 아래 | 시전 후 지속적 효과 유지 |
| **반복** (습관) | 점 3개 세로 (⋮) | 룬 우측 | 주기적 반복 발동 |
| **완료** (과거) | 작은 원 (○) | 룬 위 | 시전 완료 확정, 취소 불가 |
| **조건부** (가정) | 작은 삼각 (△) | 룬 앞(좌) | 특정 조건 충족 시에만 발동 |

#### 4) 부정과 반전 (Negation & Inversion)

| 부정 유형 | 그리기 | 위치 | 효과 | 예시 |
|-----------|--------|------|------|------|
| **부정** (않다/not) | 짧은 가로선 | 룬 위 가로지름 | 효과 무효화 | `—열기` = 불이 꺼짐 (소화) |
| **반전** (반대) | 작은 X | 룬 위 | 효과 정반대 전환 | `X열기` = 냉기, `X이사` = 용해 |
| **흡수** (받다) | 작은 ∩ | 룬 아래 | 대상 원소를 흡수/무효화 | `∩열기` = 불 흡수 (내화) |

#### 5) 조사 판정 규칙

1. **크기**: 메인 룬 bbox의 **10% 이하** → 조사로 판정
2. **거리**: 메인 룬 bbox에서 **15% 이내** 거리 → 해당 룬에 부착
3. **우선순위**: 강도 부사 > 격조사 > 시제 > 부정 (겹치면 우선순위 높은 것만 적용)
4. **복합 조사**: 격조사 + 강도 부사는 동시 적용 가능 (예: 주격 + 극대 = "불이 **몹시** 솟구친다")

#### 조사 종합 예시

```
문장: ·열기(△) ‖ + 케나즈(<) + 이사(|)· ~

해석:
  ·열기(△) ‖  = 주격(·) + 극대(‖) → "불이 맹렬히"
  케나즈(<)    = 접속사(지향) → "~를 향해"
  이사(|)· ~   = 목적격(·) + 지속(~) → "얼음을 지속적으로"

= "불이 맹렬히 얼음을 향해 지속적으로 타오른다"
→ 효과: 극대 화염 투사, 지속형, 얼음 대상 특효
→ 위력 ×2.0, 불안정성 +25
```

## 12. 문장 규칙 — 읽기 순서와 다중 룬 문장 (Sentences)

### 읽기 순서 (Reading Direction)

룬의 공간 배치 방향에 따라 마법의 **의도**가 결정됩니다:

| 배치 방향 | 의미 | 마법 성질 |
|-----------|------|----------|
| ← 왼→오 → | **투사** (Projecting) | 공격적, 대상을 향해 발사 |
| ← 오→왼 → | **흡수** (Absorbing) | 방어적, 에너지를 끌어당김 |
| ↓ 위→아래 | **하강** (Descending) | 접지, 안정화, 무게 부여 |
| ↑ 아래→위 | **상승** (Ascending) | 비상, 강화, 초월 |
| ↻ 시계방향 | **증폭** (Amplifying) | 원형 배열 시 마력 증가 |
| ↺ 반시계방향 | **소멸** (Draining) | 원형 배열 시 에너지 흡수/봉인 |

### 다중 룬 문장 등급

| 룬 수 | 문장 등급 | 특성 | 불안정성 기본 보정 |
|--------|-----------|------|--------------------|
| 1 | **단어** (Word) | 단일 효과 | ±0 |
| 2 | **구** (Phrase) | 부수 조합 (현재 시스템) | 조합표 참조 |
| 3 | **절** (Clause) | 삼원소 상호작용, 복합 효과 | +15 |
| 4 | **문장** (Sentence) | 완전한 술식, 자기 유지 가능 | +30 |
| 5+ | **주문** (Incantation) | 고위 마법, 극도로 불안정 | +50 |

### 문장 구조 패턴 (Sentence Patterns)

자주 쓰이는 다중 룬 조합 패턴:

#### 패턴 1: 주어-동사 (Subject-Verb)
```
[원소 룬] + [행위 룬]
예: 열기(△) + 티와즈(↑) = "불이 솟구친다" (화염 투사)
```

#### 패턴 2: 주어-동사-대상 (S-V-O)
```
[원소 룬] + [행위 룬] + [대상 룬]
예: 이사(|) + 케나즈(<) + 열기(△) = "얼음이 불을 향해 간다" (얼음 투사→불)
```

#### 패턴 3: 조건-결과 (If-Then)
```
[조건 룬] + 이사(|) + [결과 룬]  (이사를 접속사 THEN으로 사용)
예: 알기즈(Y) + | + 소울로(⚡) = "방어 후 번개 반격"
```

#### 패턴 4: 봉인 구조 (Seal)
```
뼈대 원(○) 안에 [룬 A] + 나우디즈(+) + [룬 B]
예: ○{ 열기(△) + 이사(|) } = "불과 얼음을 원 안에 속박" (상쇄 봉인)
```

#### 패턴 5: 순환 강화 (Cycle)
```
뼈대 원(○) 위에 같은 룬 3개 + 핵 룬 1개
예: ○[소울로·소울로·소울로] + 핵:원(○) = "번개 삼중 순환 → 플라즈마 구 강화"
```

### 복합 뼈대-룬 문장 예시

| 구조 | 구성 | 효과 이름 | 설명 |
|------|------|-----------|------|
| △{열기} ─ 뼈대직선 ─ ○{이사} | 삼각 가두기 + 직선 연결 + 원 가두기 | **화빙 쌍포** | 삼각에서 불 집중 → 원에서 얼음 봉인, 교대 발사 |
| □{알기즈 + 열기} | 사각 가두기 + 속박 | **화염결계** | 결계 내부에 화염 방벽 지속 유지 |
| ○[티와즈·티와즈·티와즈] | 원형 배열, 같은 룬 3개 | **창우** (Spear Rain) | 삼중 투창이 순환 강화 후 전방위 발사 |
| 열기 ㅡ 라구즈 ◇ 소울로 | 병렬 + 전환 | **원소 연쇄** | 불+물 동시 → 증기로 전환 → 번개 |
| ○{◇{나우디즈}} | 이중 가두기 (원+마름모) | **절대 봉인** | 이중 봉인으로 극한 억제, 불안정성 극저 |

> **구현 상태**: 9~12장의 문장 규칙은 **설계 명세(spec)** 단계입니다.
> 현재 코드에 구현된 것은 2장(부수 조합)과 3장(뼈대 수식어)의 기본 형태뿐입니다.
